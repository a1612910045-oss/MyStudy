# 场景管理与加载
> 2026-06 | Unity · SceneManager · 异步加载 · 场景叠加

场景是 Unity 项目的基本组织单位。简单项目一个场景就够了，但稍微上规模的项目就涉及场景切换、叠加、异步加载和过渡动画。

## 基本操作

```csharp
using UnityEngine.SceneManagement;

// 同步加载——会卡帧，仅用于原型
SceneManager.LoadScene("GameScene");

// 异步加载（推荐）
SceneManager.LoadSceneAsync("GameScene");
```

两个 API 看似相似，但同步版 `LoadScene` 会阻塞主线程。场景一大，玩家会看到明显卡顿。生产代码一律用异步版。

## 异步加载的完整流程

```csharp
IEnumerator LoadSceneCoroutine(string sceneName)
{
    // 1. 显示加载界面
    loadingUI.SetActive(true);

    // 2. 开始异步加载
    var asyncOp = SceneManager.LoadSceneAsync(sceneName);
    asyncOp.allowSceneActivation = false; // 加载完不立即切换

    // 3. 更新进度条
    while (asyncOp.progress < 0.9f)
    {
        float progress = asyncOp.progress / 0.9f;
        loadingBar.fillAmount = progress;
        yield return null;
    }

    // 4. 进度 0.9 意味着资源已加载完毕，可以激活
    loadingBar.fillAmount = 1f;
    yield return new WaitForSeconds(0.5f); // 给玩家看一眼满进度

    // 5. 激活场景
    asyncOp.allowSceneActivation = true;
}
```

关键点：
- `allowSceneActivation = false` 时进度停在 0.9——这是 Unity 有意设计的分界点。0-0.9 是资源加载，激活切换是最后一步。
- `LoadSceneMode` 默认是 `Single`（替换当前场景）。

## 场景叠加（Multi-Scene）

`Single` 模式会卸载当前场景，`Additive` 模式在当前场景上叠加：

```csharp
SceneManager.LoadSceneAsync("UIScene", LoadSceneMode.Additive);
```

常见模式：
- **主场景（常驻）**：包含 GameManager、EventSystem、常驻 UI 等。
- **关卡场景**：按需加载，切换时卸载旧关卡加载新关卡。
- **UI 场景**：独立场景，不受关卡切换影响。

用 `SceneManager.UnloadSceneAsync` 卸载叠加的场景：

```csharp
SceneManager.UnloadSceneAsync("Level01");
```

## DontDestroyOnLoad

某些对象需要跨场景存活——比如音乐播放器、网络管理器、全局事件系统：

```csharp
void Awake()
{
    DontDestroyOnLoad(gameObject);
}
```

但 DontDestroyOnLoad 要小心使用：
- 如果反复进出场景，每次都创建新实例，会导致重复对象堆积。配合单例模式：

```csharp
public class AudioManager : MonoBehaviour
{
    public static AudioManager Instance { get; private set; }

    void Awake()
    {
        if (Instance != null)
        {
            Destroy(gameObject);
            return;
        }
        Instance = this;
        DontDestroyOnLoad(gameObject);
    }
}
```

## Active Scene

当多个场景叠加时，`SceneManager.GetActiveScene()` 返回的是「活跃场景」。新实例化的对象默认放在活跃场景中。可以用 `SceneManager.SetActiveScene()` 切换：

```csharp
SceneManager.SetActiveScene(SceneManager.GetSceneByName("Level01"));
```

## 场景加载事件

```csharp
void OnEnable()
{
    SceneManager.sceneLoaded += OnSceneLoaded;
    SceneManager.sceneUnloaded += OnSceneUnloaded;
}

void OnSceneLoaded(Scene scene, LoadSceneMode mode)
{
    Debug.Log($"场景加载完成: {scene.name}, 模式: {mode}");
}

void OnDisable()
{
    SceneManager.sceneLoaded -= OnSceneLoaded;
    SceneManager.sceneUnloaded -= OnSceneUnloaded;
}
```

## 性能与优化

- 场景加载时，Unity 会初始化场景中所有 GameObject（Awake → OnEnable）。大量对象同时初始化可能导致 CPU 尖峰。
- 对于复杂关卡，把大场景拆成子场景，用 Additive 分帧加载。
- Addressables 支持异步加载场景，走的是同样的 SceneManager 但资源管理更精细。
- 场景切换时的 GC Spike：旧场景卸载可能导致大量对象被 GC。用 `System.GC.Collect()` 在加载界面期间主动回收（谨慎使用，仅在确实需要时）。

## 微信小游戏特殊限制

微信小游戏（WebGL）场景加载有几个限制：
- 不能同步加载场景（`LoadScene` 会卡死），必须用异步。
- WebGL 不支持多线程，场景加载在主线程执行，大场景可能引起明显停顿。
- 建议场景尽量小、预制体尽量复用、用 Addressables 按需加载而不是整场景打包。
