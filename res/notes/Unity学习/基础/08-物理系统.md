# 物理系统
> 2026-06 | Unity · 物理 · Rigidbody · Collider · Raycast

Unity 内置两套物理引擎：3D 物理（PhysX）和 2D 物理（Box2D）。两套 API 几乎是对称的，命名加 `2D` 后缀即可（`Rigidbody` → `Rigidbody2D`）。以下以 2D 为例，3D 同理。

## Rigidbody：让对象参与物理

Rigidbody 是物理系统的核心。没有 Rigidbody 的对象，就算有 Collider，也不会自动受重力和碰撞力的影响。

```csharp
private Rigidbody2D rb;

void Awake()
{
    rb = GetComponent<Rigidbody2D>();
}

void FixedUpdate()
{
    rb.velocity = new Vector2(moveInput.x * speed, rb.velocity.y);
}
```

关键属性：

| 属性 | 说明 |
|---|---|
| Body Type (simulated) | Dynamic（受力）/ Kinematic（不受力但可移动）/ Static（不动） |
| Gravity Scale | 重力倍率，0 为无重力 |
| Mass | 质量 |
| Drag / Angular Drag | 线性/旋转阻力 |

**移动方式对比**：

```csharp
// 1. 直接修改 velocity（推荐，物理模拟友好）
rb.velocity = new Vector2(speed, rb.velocity.y);

// 2. 加力（模拟推动，有惯性）
rb.AddForce(Vector2.right * force, ForceMode2D.Impulse);

// 3. 修改 position（跳过了物理引擎，可能穿墙）
rb.MovePosition(rb.position + velocity * Time.fixedDeltaTime);
```

不要用 `transform.position` 移动有 Rigidbody 的对象——它跳过了物理引擎，会导致碰撞检测失败和穿墙。

## Collider：碰撞体

Collider 定义物理形状。没有 Rigidbody 也可以有 Collider（作为静态障碍物）。有 Rigidbody 时，Collider 决定碰撞边界。

常见类型：BoxCollider2D、CircleCollider2D、CapsuleCollider2D、PolygonCollider2D、EdgeCollider2D。

关键属性：
- **Is Trigger**：勾选后碰撞不产生物理反馈，只触发 `OnTriggerEnter` 事件（用于检测区域、道具拾取）。
- **Physics Material 2D**：控制弹力和摩擦系数。

## 碰撞检测的四个回调

```csharp
// 碰撞（有物理反馈）
void OnCollisionEnter2D(Collision2D collision) { }
void OnCollisionStay2D(Collision2D collision) { }
void OnCollisionExit2D(Collision2D collision) { }

// 触发器（无物理反馈）
void OnTriggerEnter2D(Collider2D other) { }
void OnTriggerStay2D(Collider2D other) { }
void OnTriggerExit2D(Collider2D other) { }
```

`OnCollisionEnter2D` 的参数是 `Collision2D`，包含接触点、相对速度等详细碰撞信息。`OnTriggerEnter2D` 的参数只是对方 Collider。

碰撞/触发的条件是 **双方都有 Collider，且至少一方有 Rigidbody**。

## Raycast：射线检测

```csharp
// 2D 射线
RaycastHit2D hit = Physics2D.Raycast(origin, direction, distance, layerMask);

if (hit.collider != null)
{
    Debug.Log($"命中: {hit.collider.name}，距离: {hit.distance}");
}

// 或者直接检测多个结果
RaycastHit2D[] hits = new RaycastHit2D[10];
int count = Physics2D.RaycastNonAlloc(origin, direction, hits, distance, layerMask);
for (int i = 0; i < count; i++)
{
    ProcessHit(hits[i]);
}
```

`RaycastNonAlloc` 用预分配数组避免了 GC——在频繁调用的场景（每帧检测地板、墙面）推荐。

3D 版本：`Physics.Raycast`（返回 bool + out hit）。

## LayerMask：过滤碰撞

Layer 可以在 `Edit → Project Settings → Physics 2D → Layer Collision Matrix` 中配置哪些层之间发生碰撞（物理层面）。代码中也可以手动过滤：

```csharp
// 只检测 "Ground" 和 "Platform" 层
int layerMask = LayerMask.GetMask("Ground", "Platform");
RaycastHit2D hit = Physics2D.Raycast(origin, Vector2.down, 1f, layerMask);

// 排除自身层
int inverseMask = ~LayerMask.GetMask("Player");
```

## 常见问题

**角色抖动**：通常是因为同时用了物理移动和 Transform 修改。解决：统一使用 Rigidbody API（MovePosition / velocity），并在 FixedUpdate 里操作。

**穿墙**：
- 静态物体设为 Static，移动物体设为 Dynamic。
- Collision Detection 设 `Continuous`（高速度物体）。
- 减小 Fixed Timestep（Project Settings → Time → Fixed Timestep）。

**OnTrigger 不触发**：检查双方是否都有 Collider，至少一方有 Rigidbody；Is Trigger 必须勾选；检查 Layer Collision Matrix 是否允许交互。

**性能优化**：
- 尽量用简单碰撞体（Box/Circle/Capsule），避免 MeshCollider 和复杂 PolygonCollider。
- 静态地图用 TilemapCollider2D 或 CompositeCollider2D——多个小块合并成一个碰撞体。
- 大量物体使用 Physics2D.Simulate 控制更新频率。
