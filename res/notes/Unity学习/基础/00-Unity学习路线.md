# Unity 学习路线
> 2026-06 | Unity · 学习路线 · 游戏开发

这套笔记面向有一年左右 Unity 实际项目经验的开发者。目标不是从零入门，而是把零散经验串成体系：知道为什么这么做、有哪些替代方案、什么时候该用哪种。

## 学习目标

- 理解 Unity 编辑器的设计意图：场景、预制体、资源、包管理器各自解决什么问题。
- 掌握 GameObject / Component 架构的本质——它不是继承树，而是组合模式。
- 熟练运用 MonoBehaviour 生命周期，知道每帧和事件性回调的区别。
- 能用 UGUI 搭建响应式界面，理解 Canvas 三种渲染模式和 RectTransform 锚定。
- 理解 Animator 状态机、BlendTree 和动画事件，能拆分动画逻辑。
- 能用协程处理分帧逻辑，但知道它和 C# 异步不是一回事。
- 掌握 Addressables 资源管理，替代 Resources 和 AssetBundle 的历史方案。
- 理解性能优化的核心指标：DrawCall、GC、CPU/GPU 瓶颈、Profiler 使用。
- 知道微信小游戏打包的特殊限制和适配策略。

## 推荐顺序

1. 编辑器与项目结构——先把「地盘」摸清楚。
2. GameObject 与 Component——理解 Unity 的架构哲学。
3. MonoBehaviour 生命周期——脚本运行的节奏。
4. Transform 与坐标系——空间操作基础。
5. 输入系统——从旧 Input 到新 Input System。
6. UGUI 与界面——做 UI 逃不掉的知识。
7. 动画系统——Animator 状态机实战。
8. 物理系统——刚体、碰撞与射线。
9. 资源管理——Addressables 实战。
10. 场景管理——加载、卸载与过渡。
11. Spine 骨骼动画——2D 项目的动画管线。
12. 协程与异步——分帧执行与等待。
13. 性能优化——Profiler 驱动的优化流程。

## 怎么学

- 每篇笔记对应一个实战问题。先遇到问题，再回来看笔记。
- 不要背 API。理解设计意图比记住参数更重要。
- 每个概念都动手验证一次——在空项目里建场景、挂脚本、跑起来。
- Unity 官方文档和 Scripting API 是最好的参考，这些笔记是「带路的地图」。

## 一个贯穿练习

做一个「打砖块」小游戏，贯穿大部分知识点：

- 场景搭建 + 预制体（砖块、球、挡板）
- 刚体物理 + 碰撞检测
- UGUI 分数显示和菜单
- 场景切换（开始菜单 → 游戏 → 结算）
- Addressables 加载音效和贴图
- 用动画系统做砖块破碎效果
- 用 Profiler 检查性能热点

这个练习不复杂，但能覆盖 80% 的实际开发场景。
