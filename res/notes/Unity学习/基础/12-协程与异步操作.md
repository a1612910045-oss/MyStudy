# 协程与异步操作
> 2026-06 | Unity · 协程 · async · await · UniTask

Unity 里有三种处理「等待」的方式：协程、async/await、Update 计时器。它们解决的问题相似，但原理完全不同。

## 协程的本质

协程是把一个方法分到多个帧里执行。它不是多线程——所有逻辑都在主线程按顺序跑，只是在 `yield return` 处暂停，等条件满足后继续。

```csharp
IEnumerator FadeOut(Image img, float duration)
{
    float elapsed = 0f;
    Color color = img.color;

    while (elapsed < duration)
    {
        elapsed += Time.deltaTime;
        color.a = 1f - (elapsed / duration);
        img.color = color;
        yield return null; // 暂停，下一帧继续
    }

    img.gameObject.SetActive(false);
}

void Start()
{
    StartCoroutine(FadeOut(uiImage, 1f));
}
```

## yield return 的几种类型

| yield 类型 | 含义 |
|---|---|
| `yield return null` | 等一帧 |
| `yield return new WaitForSeconds(n)` | 等待 n 秒（受 Time.timeScale 影响） |
| `yield return new WaitForSecondsRealtime(n)` | 等待 n 秒（不受 timeScale 影响） |
| `yield return new WaitForEndOfFrame()` | 等当前帧渲染完成 |
| `yield return new WaitUntil(() => condition)` | 等条件为真 |
| `yield return StartCoroutine(other)` | 等另一个协程执行完 |
| `yield return asyncOperation` | 等异步操作完成（加载场景等） |

**避免在循环里 new WaitForSeconds**——每次 yield 都 new 一个会制造 GC：

```csharp
// 不好的写法
while (true)
{
    yield return new WaitForSeconds(1f); // 每秒分配一个对象
}

// 好的写法
private WaitForSeconds waitOneSecond = new WaitForSeconds(1f);
while (true)
{
    yield return waitOneSecond; // 复用
}
```

## 协程的生命周期

协程绑定到启动它的 MonoBehaviour：
- GameObject 失活 → 协程暂停。
- 组件被 Disable → 协程不会停止！这是个常见坑。Disable 组件不会 Stop 协程，除非你手动 Stop。
- GameObject 被 Destroy → 所有协程自动停止。

手动停止协程：

```csharp
Coroutine fadeCoroutine = StartCoroutine(FadeOut());

// 停止特定协程
StopCoroutine(fadeCoroutine);

// 停止该脚本上所有协程
StopAllCoroutines();
```

## 协程与异常

协程里的异常不会传播到启动协程的代码，也不会被外层的 try-catch 捕获——它们直接打到 Unity 的日志里。

```csharp
StartCoroutine(BuggyCoroutine());
Debug.Log("这行会执行——异常不会向上传播");

IEnumerator BuggyCoroutine()
{
    throw new System.Exception("协程内的异常,直接打到 Console");
    yield return null;
}
```

要在协程里安全地处理异常，只能在协程内部 try-catch。

## async / await 在 Unity 中

C# 的 `async` / `await` 在 Unity 里也可以用，但有局限性：

```csharp
async void Start()
{
    var text = await LoadConfigAsync();
    Debug.Log(text);
}

async Task<string> LoadConfigAsync()
{
    var request = UnityWebRequest.Get(url);
    await request.SendWebRequest();
    return request.downloadHandler.text;
}
```

`await request.SendWebRequest()` 是在底层用协程实现的——它把异步操作挂到 Unity 的更新循环里，不会阻塞主线程。

但原生 async/await 缺少超时、取消、帧级控制等 Unity 项目必需的功能。这就是 **UniTask** 存在的理由。

## UniTask

UniTask 是 Cysharp 开源的零分配 async/await 库，专为 Unity 设计。

```csharp
// 安装：Package Manager → Add from git URL:
// https://github.com/Cysharp/UniTask.git?path=src/UniTask/Assets/Plugins/UniTask

async UniTaskVoid Start()
{
    // 等待秒数（不受 timeScale 影响）
    await UniTask.Delay(TimeSpan.FromSeconds(2));

    // 等待帧数
    await UniTask.DelayFrame(3);

    // 等到下一帧
    await UniTask.Yield();

    // 等待条件
    await UniTask.WaitUntil(() => hp > 0);

    // 带取消
    var cts = new CancellationTokenSource();
    await SomeAsync(cancellationToken: cts.Token);

    // 超时
    var result = await SomeAsync().Timeout(TimeSpan.FromSeconds(3));
}
```

UniTask 的优势：
- 零 GC 分配（基于 struct 的 async 方法构建器）。
- 完整支持取消令牌（CancellationToken）。
- 可以指定 `PlayerLoopTiming`（在哪个生命周期阶段恢复执行）。
- 提供 `UniTaskTracker` 窗口，调试运行中的异步任务。
- 比协程更接近 C# 标准，方便跨项目复用。

## 什么时候用什么

| 场景 | 推荐方案 |
|---|---|
| 简单分帧逻辑（< 5 行） | 协程 |
| 有多个 await 的复杂流程 | UniTask |
| 网络请求、文件 IO | UniTask |
| 与第三方 .NET 库交互 | UniTask / async Task |
| UI 过渡动画链 | 协程（配合 DOTween 更简洁） |
| 需要超时/取消的等待 | UniTask |

纯协程在复杂异步流程里容易写出「回调地狱」式的嵌套 StartCoroutine。UniTask 用 `await` 让它回归线性流程。

## 协程和 UniTask 的配合

你可以在 UniTask 里等协程：

```csharp
async UniTask DoSomething()
{
    // 等一个协程完成
    await SomeCoroutine().ToUniTask();
}

IEnumerator SomeCoroutine()
{
    yield return new WaitForSeconds(1f);
}
```
