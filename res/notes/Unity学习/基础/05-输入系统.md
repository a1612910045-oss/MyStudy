# 输入系统
> 2026-06 | Unity · Input · Input System · 键盘鼠标 · 触屏

Unity 有两套输入方案：旧的 `Input` 类（Input Manager）和新的 Input System 包。老项目可能还在用旧的，但新项目建议直接从 Input System 开始。

## 旧版 Input（Input Manager）

放在 `Edit → Project Settings → Input Manager`。本质上是键位名称到按键的映射。

```csharp
void Update()
{
    float horizontal = Input.GetAxis("Horizontal");   // -1 ~ 1，平滑
    float rawHorizontal = Input.GetAxisRaw("Horizontal"); // -1, 0, 1，无平滑

    if (Input.GetButtonDown("Jump"))     // 按下瞬间
    {
        Jump();
    }

    if (Input.GetKey(KeyCode.W))        // 按住
    {
        MoveForward();
    }
}
```

优点：简单，API 直接，学习成本低。适合 jam 和小型项目。

缺点：
- 不支持手柄热插拔和多人输入区分。
- 触屏操作需要额外处理。
- 所有输入逻辑混在 Update 里，难以复用。

## 新版 Input System

需通过 Package Manager 安装 `com.unity.inputsystem`。核心变化：输入不再是「读按键」，而是「订阅事件」。

### 安装与配置

安装后在 Player Settings 里将 Active Input Handling 设为 `Input System Package (New)`。可以同时支持新旧两套（`Both`），但不推荐长期混用。

安装时会提示生成 `.inputactions` 资源文件，这就是你的输入配置。

### Action 的三种事件

```csharp
public class PlayerInput : MonoBehaviour
{
    private PlayerControls controls;

    void Awake()
    {
        controls = new PlayerControls();
    }

    void OnEnable()
    {
        controls.Gameplay.Enable();

        // 执行中（每帧都有值）
        controls.Gameplay.Move.performed += ctx => OnMove(ctx);

        // 按键开始
        controls.Gameplay.Jump.started += _ => StartJump();

        // 按键松开
        controls.Gameplay.Jump.canceled += _ => StopJump();
    }

    void OnDisable()
    {
        controls.Gameplay.Disable();
    }

    void OnMove(InputAction.CallbackContext ctx)
    {
        Vector2 input = ctx.ReadValue<Vector2>();
        // 移动逻辑
    }
}
```

三种回调语义：
- `started`：交互刚开始（手指碰到屏幕、按键按下）。
- `performed`：满足触发条件（按住一定时间或到达阈值）。
- `canceled`：交互结束。

### PlayerInput 组件

Input System 还提供了 `PlayerInput` 组件，可以通过 Inspector 拖拽绑定 Action Map，然后用 UnityEvent 或 `SendMessage` 调用你的方法。

自动设备检测和多人分屏全靠它。要快速上手，直接给 GameObject 挂 `PlayerInput` 组件，Behavior 选 `Invoke Unity Events`，然后把各个 Action 的事件拖到你的方法上。

### 触屏和移动端

Input System 原生支持触屏：

```csharp
// 读取第一个触摸点
var touch = Touchscreen.current?.primaryTouch;
if (touch?.press.isPressed ?? false)
{
    Vector2 pos = touch.position.ReadValue();
}
```

对于摇杆类操作，推荐在 Actions 里直接定义 `Stick` 类型的 Action，然后把它的 `performed` 绑定到移动方法。

## 迁移策略

- 新项目直接用 Input System，不用犹豫。
- 老项目：先全局搜 `Input.Get` 统计使用量。量小（< 20 处）直接全替换；量大按模块逐步切。
- 可以新旧混用（Both），但只在过渡期这样做。长期混用增加调试复杂度。

## 最佳实践

- 把输入读取和业务逻辑分开：输入层只负责「玩家想做什么」，不要直接在 `performed` 回调里写游戏逻辑。
- 用 ScriptableObject 做输入事件通道：输入系统触发事件 → SO 事件 → 多个订阅者响应。解耦效果非常好。
- 禁用未使用的 Action Map：菜单场景里禁用 Gameplay Map，减少意外输入。
- 在 `OnEnable` / `OnDisable` 里管理 Action Map 的启用/禁用——和对象生命周期保持一致。
