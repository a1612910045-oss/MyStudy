# 动画系统
> 2026-06 | Unity · Animator · Animation · AnimatorController

Unity 的动画系统分两层：底层是 Animation（动画片段），上层是 Animator（状态机）。绝大多数业务逻辑只需要用到 Animator，但理解底层有助于调试。

## Animation Clip

动画片段是属性随时间变化的曲线集合。它可以驱动 Transform、SpriteRenderer、Image 等任何 Component 的序列化属性。

在 Animation 窗口里录制时，Unity 会自动生成 `.anim` 文件和一个 Animator Controller。你可以选择「录制模式」逐帧调，也可以直接拖动精灵序列到场景做帧动画。

关键属性：
```csharp
// 播放一段动画（简单用法）
animation.Play("Walk");

// 检查是否在播放某动画
animation.IsPlaying("Attack");
```

但实际项目中几乎不用 `Animation` 组件，而用 `Animator`——因为状态机更灵活。

## Animator 状态机

Animator = AnimatorController（状态机） + 参数 + 动画片段。

### 四个概念

- **State**：每个状态通常绑定一个 Animation Clip。默认状态是橙色（Entry → 指向）。
- **Transition**：状态之间的连线。Has Exit Time 决定是否等当前动画播完再切。
- **Parameter**：Bool、Int、Float、Trigger。脚本通过设置参数来控制状态跳转。
- **Layer**：遮罩层。比如上半身射击 + 下半身跑动。

### 脚本控制

```csharp
private Animator anim;

void Awake()
{
    anim = GetComponent<Animator>();
}

void Update()
{
    // 设置参数驱动动画
    anim.SetBool("IsMoving", isMoving);
    anim.SetFloat("Speed", speed);
    anim.SetTrigger("Attack");
    anim.SetInteger("State", stateIndex);
}
```

Trigger 是一次性信号——设置一次后自动消费，不需要手动复位。Bool/Float/Int 是持续状态，适合速度、方向等连续变化量。

### BlendTree

BlendTree 让一个参数驱动多个动画之间的混合。最常见的用法：用 Speed 参数在 Idle / Walk / Run 之间平滑过渡。

```
BlendTree "Locomotion"
  ├── Idle   (Speed = 0)
  ├── Walk   (Speed = 0.5)
  └── Run    (Speed = 1.0)
```

1D BlendTree 用于直线混合，2D BlendTree 用于方向混合（四方向/八方向移动）。

创建 BlendTree：在 Animator 窗口右键 → Create State → From New BlendTree → 双击进入 → 添加 Motion Field → 拖入各动画片段并设置阈值。

### Animation Event

动画片段上可以插入事件，在指定帧回调脚本方法：

```csharp
// 在 Attack 动画的挥砍关键帧上插入事件 "OnSwing"
void OnSwing()
{
    // 检测命中
}

// 脚步声事件
void OnFootstep()
{
    AudioSource.PlayOneShot(footstepSound);
}
```

事件方法和 Animation Clip 必须在同一个 GameObject 上。如果动画在子对象上触发事件，Unity 会向上递归查找父对象上的同名方法（通过 `SendMessage`，性能较低）。推荐把事件放在 Animator 所在的同一 GameObject 上。

## 动画层级

Layer 配合 Avatar Mask 可以实现局部动画覆盖：

```
Base Layer: 全身（Idle / Walk / Run）
Upper Layer: 仅上半身（Shoot），权重 1，Blending = Override
```

Layer 的 Weight 可以在脚本中动态调整：

```csharp
anim.SetLayerWeight(upperLayerIndex, isAiming ? 1f : 0f);
```

## 性能注意事项

- Animator 即使没有可见变化，每帧也会更新状态机。不需要动画的对象，不要挂着 Animator（可以 Disable，或直接移除）。
- `Animator.StringToHash`：参数查找用哈希比字符串更快。

```csharp
private static readonly int IsMovingHash = Animator.StringToHash("IsMoving");

anim.SetBool(IsMovingHash, true); // 比 SetBool("IsMoving", true) 快
```

- 复杂的状态机（几十个状态 + 上百个 Transition）在移动端可能有明显的 CPU 开销。考虑简化或改用代码驱动的动画。
- Animator Controller 的 `Update Mode` 设 `Unscaled Time` 影响的是 Animator 自身的时间尺度，不是 Time.timeScale。暂停游戏时 UI 动画需要设置它。

## Spine 与 Unity 动画的配合

如果项目用 Spine（2D 骨骼），常规做法是：
- 角色的位移/翻转由 Unity Transform 控制。
- 角色动作（Idle、Walk、Attack）由 Spine 的 SkeletonAnimation 控制。
- 不要试图把 Spine 动画放进 Unity Animator——它们是完全不同的系统。

Spine 的动画切换可以直接通过 `skeletonAnimation.AnimationState.SetAnimation(0, "walk", true)` 实现，不需要通过 Unity Animator。
