# 微信小游戏适配
> 2026-06 | Unity · 微信小游戏 · WebGL · SDK 接入

微信小游戏是国内游戏开发的重要分发渠道。从 Unity 打包到微信小游戏，需要经历平台适配、SDK 接入、性能调优三个主要环节。

## 技术栈概述

Unity 打包微信小游戏的路线是：

```
Unity C# → IL2CPP → WebAssembly → 微信小游戏运行时
```

微信小游戏本质上是 WebGL 平台的封装。Unity 通过专有的 WebGL 构建目标 + 微信小游戏转换工具（插件），把 WebGL 产物包进微信的 `.wxapkg` 格式。

核心依赖：
- **Unity WebGL 构建模块**：必须安装。
- **微信小游戏转换插件**：把 WebGL 产物转为微信可用格式。
- **Unity 微信 SDK**：提供微信登录、支付、分享、广告等 JS 桥接。

## 项目配置

### Player Settings 关键调整

```
Player Settings:
├── Resolution and Presentation
│   ├── Default Canvas Width: 750
│   ├── Default Canvas Height: 1334
│   └── Run In Background: ✓（切后台不暂停）
├── Other Settings
│   ├── Color Space: Linear（Gamma 也行，但 Linear 色彩更好）
│   ├── Auto Graphics API: ✓
│   ├── Strip Engine Code: ✓（裁减没用到的引擎代码，减小包体）
│   ├── Managed Stripping Level: Medium / High
│   └── IL2CPP Code Generation: Faster (smaller) runtime
├── Publishing Settings
│   ├── Compression Format: Gzip
│   └── Enable Exceptions: None（或 Explicitly Thrown Only）
```

异常处理设为 `None` 可显著减小 WebAssembly 体积，但会导致 try-catch 失效。如果必须捕获异常，选 `Explicitly Thrown Only`。

### 包体控制

微信小游戏对首包（首次下载的体积）有限制。目前大体是：
- 首次启动下载：≤ 20MB
- 分包加载：每个分包 ≤ 20MB，总数看类目

策略：
- 首包只放启动必需的资源：Logo、Loading 界面、主界面。
- 关卡资源、音效、大图等打到分包，运行时按需下载。
- Addressables 完美适配这个模型——首包 Group 设为 LocalBuildPath，分包 Group 设为 RemoteBuildPath。

## SDK 接入

### 微信原生 API 调用

通过 JS 桥接调用微信 SDK：

```csharp
// C# 端
[DllImport("__Internal")]
private static extern void WXLogin();

public void Login()
{
#if UNITY_WEBGL && !UNITY_EDITOR
    WXLogin();
#endif
}
```

大多数项目会用封装好的 Unity-WebGL-微信插件，不需要手写 `DllImport`。常见的功能：
- 微信登录 / 授权
- 分享（`wx.shareAppMessage`）
- 激励视频广告
- Banner 广告
- 支付（`wx.requestMinePayment`）

## 常见适配问题

### 文件系统

WebGL 平台没有传统的文件系统。`Application.persistentDataPath` 实际映射到 IndexedDB（浏览器数据库），容量和性能有限。

- `PlayerPrefs`：底层也是 IndexedDB，只适合存少量配置。
- 大量数据用 `Application.persistentDataPath` + `File.WriteAllBytes`，但要关注存储空间限制。
- 微信提供 `wx.getFileSystemManager()` 用于文件读写，但需要通过 JS 桥接。

### 网络请求

- Unity 的 `UnityWebRequest` 在 WebGL 里底层调的是浏览器的 `XMLHttpRequest` / `fetch`。
- 微信小游戏限制了请求域名——必须在微信后台配置 `request 合法域名`。
- WebSocket 也需要配置 `socket 合法域名`。
- HTTPS 是强制要求。

### 音频

- WebGL 的音频支持比较弱。`AudioSource.PlayOneShot` 在移动端可能有延迟或失败。
- 微信提供 `wx.createInnerAudioContext()` 作为替代方案，适合播放长音频（BGM）。
- 短音效（SFX）尽量用小文件和 WebAudio API，而不是 Unity 的 AudioSource。
- 微信小游戏要求在用户交互后才能播放音频（浏览器自动播放策略）。

### 输入

- 微信小游戏中的文本输入需要调用 `wx.showKeyboard()`，Unity 原生 InputField 可能不工作。
- 第三方输入框插件或自绘输入框是常见方案。

### 字体

- Unity 默认字体（Arial）在 WebGL 中可能渲染异常。
- 使用 TextMeshPro + SDF 字体是更可靠的选择。
- 中文字体文件大。如需动态文本，使用系统默认字体（`Resources.GetBuiltinResource<Font>("LegacyRuntime.ttf")`）或裁剪后的 TMP SDF 字体。

## 调试

- **Chrome DevTools**：微信开发者工具本质是定制的 Chromium。右键 → 审查元素，可以看到 Console、Network、Sources。
- **真机调试**：微信开发者工具 → 预览 → 扫码。真机上行为可能与模拟器不同（特别是音频、性能、存储）。
- **vConsole**：微信内置的调试面板，可以在真机上查看日志和网络请求。

## 提交审核的注意事项

- 类目和资质：游戏需要在微信公众平台选择正确类目，部分类目需要版号。
- 用户隐私协议：必须展示隐私政策。
- 首次授权：获取用户信息前需要用户明确同意。
- 内容安全：文本和图片都走内容安全审核。
- 测试账号：提供可完整体验游戏的测试账号，审核通过率会高很多。
