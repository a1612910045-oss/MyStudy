# 集合、泛型与迭代器
> 2026-06 | C# · 泛型 · 集合

集合是业务代码最常用的工具之一。泛型让集合在复用的同时保持类型安全。你不需要记住所有集合，但必须知道常见集合的适用场景。

## List<T>

`List<T>` 是动态数组，适合按顺序存储元素：

```csharp
var items = new List<Item>();
items.Add(new Item("Potion"));
items.RemoveAt(0);
```

适合：

- 需要保持顺序。
- 经常按索引访问。
- 元素数量会变化。

不适合：

- 经常按 key 查找。
- 经常在中间插入和删除大量元素。

## Dictionary<TKey, TValue>

字典通过 key 查找 value：

```csharp
var itemMap = new Dictionary<string, Item>();
itemMap["potion"] = new Item("Potion");

if (itemMap.TryGetValue("potion", out var item))
{
    Console.WriteLine(item.Name);
}
```

推荐用 `TryGetValue`，避免 key 不存在时抛异常。

## HashSet<T>

集合用于去重和快速判断存在：

```csharp
var unlockedIds = new HashSet<string>();
unlockedIds.Add("skill_fireball");

if (unlockedIds.Contains("skill_fireball"))
{
    Console.WriteLine("Unlocked");
}
```

适合保存“是否拥有某个东西”。

## Queue<T> 与 Stack<T>

队列先进先出：

```csharp
var queue = new Queue<string>();
queue.Enqueue("A");
queue.Enqueue("B");
Console.WriteLine(queue.Dequeue()); // A
```

栈后进先出：

```csharp
var stack = new Stack<string>();
stack.Push("A");
stack.Push("B");
Console.WriteLine(stack.Pop()); // B
```

队列适合任务排队，栈适合撤销、状态回退、深度优先遍历。

## 泛型方法

泛型不只用于集合，也能用于方法：

```csharp
T PickRandom<T>(IReadOnlyList<T> source)
{
    int index = Random.Shared.Next(source.Count);
    return source[index];
}
```

`T` 代表调用时才确定的类型。

## 泛型约束

约束可以告诉编译器 T 必须满足什么条件：

```csharp
T Create<T>() where T : new()
{
    return new T();
}
```

常见约束：

- `where T : class`
- `where T : struct`
- `where T : SomeBaseClass`
- `where T : ISomeInterface`
- `where T : new()`

## IEnumerable<T> 与迭代器

`IEnumerable<T>` 表示“可以被枚举的一串东西”。

```csharp
IEnumerable<int> GetEvenNumbers(int max)
{
    for (int i = 0; i <= max; i++)
    {
        if (i % 2 == 0)
        {
            yield return i;
        }
    }
}
```

`yield return` 会生成迭代器。它不是一次性返回所有结果，而是在遍历时逐个产出。

## Unity 中的注意点

- `List<T>` 是 Unity 代码里最常见的集合。
- Inspector 对泛型和复杂集合的序列化支持有限，要了解 Unity 序列化规则。
- 热路径里频繁创建临时集合会产生 GC。
- `foreach`、LINQ、闭包是否分配，要结合 Unity 版本和实际 Profiler 判断。
