# 异步编程：async / await
> 2026-06 | C# · async · Task

异步编程用于处理“等待”：网络请求、文件 IO、数据库、延迟任务。它的目标不是让代码变快，而是在等待期间不阻塞当前线程。

## Task

`Task` 表示一个异步操作：

```csharp
Task SaveAsync();
Task<string> LoadTextAsync();
```

- `Task`：异步完成后没有返回值。
- `Task<T>`：异步完成后返回 `T`。

## async / await

```csharp
async Task<string> LoadConfigAsync(string path)
{
    string text = await File.ReadAllTextAsync(path);
    return text;
}
```

`await` 会暂停当前方法，等任务完成后继续执行。它让异步代码看起来像同步流程。

## async 方法的执行

一个 `async` 方法在遇到第一个未完成的 `await` 之前，会同步执行。遇到 `await` 后，控制权返回给调用方，任务完成后再继续。

这意味着：

```csharp
async Task DoWorkAsync()
{
    Console.WriteLine("A");
    await Task.Delay(1000);
    Console.WriteLine("B");
}
```

调用后会先打印 A，约 1 秒后打印 B。

## 不要滥用 async void

通常不要写：

```csharp
async void Load()
{
}
```

更推荐：

```csharp
async Task LoadAsync()
{
}
```

`async void` 的异常很难被调用方捕获。它主要用于事件处理器。

## 并发等待

顺序等待：

```csharp
var a = await LoadAAsync();
var b = await LoadBAsync();
```

并发等待：

```csharp
Task<string> taskA = LoadAAsync();
Task<string> taskB = LoadBAsync();

string[] results = await Task.WhenAll(taskA, taskB);
```

如果两个任务互不依赖，并发等待通常更合适。

## 取消

长任务应该支持取消：

```csharp
async Task DownloadAsync(CancellationToken token)
{
    await httpClient.GetAsync(url, token);
}
```

调用方可以通过 `CancellationTokenSource` 取消任务。

## 异常传播

异步方法里的异常会保存在 Task 中，await 时重新抛出：

```csharp
try
{
    await LoadAsync();
}
catch (Exception ex)
{
    Console.WriteLine(ex.Message);
}
```

所以不要忘记 await，否则异常可能变得难追踪。

## Unity 与 async

Unity 里可以使用 `async` / `await`，但要注意：

- Unity API 大多只能在主线程访问。
- `Task.Run` 适合 CPU 工作，但不能在后台线程操作 GameObject。
- 协程更适合分帧流程、等待帧、等待 Unity YieldInstruction。
- 异步更适合文件、网络、平台 SDK 等 IO。
