# C# 学习路线
> 2026-06 | C# · 学习路线 · .NET

这套笔记适合从“会写一点代码”过渡到“能独立写 C# 业务逻辑、读 Unity 项目代码、理解常见 .NET API”的学习路径。建议不要只背语法，而是按“语法 → 类型 → 对象模型 → 集合与泛型 → 委托事件 → LINQ → 异步 → Unity 实战”的顺序推进。

## 学习目标

- 能读懂常见 C# 代码结构：命名空间、类、字段、属性、方法、构造函数。
- 能区分值类型和引用类型，理解复制值与复制引用的差异。
- 能用类、接口、继承和组合拆分游戏或工具中的业务逻辑。
- 能熟练使用 `List<T>`、`Dictionary<TKey, TValue>`、`HashSet<T>` 等集合。
- 能用委托、事件、Lambda 表达式组织回调与消息通知。
- 能用 LINQ 做筛选、映射、分组和排序，但知道什么时候不该滥用。
- 能理解 `async` / `await` 的执行模型，知道 Unity 协程和 C# 异步不是同一套东西。
- 能写出更适合 Unity 的 C#：少分配、序列化友好、生命周期清晰。

## 推荐顺序

1. 基础语法：变量、类型、字符串、流程控制、方法。
2. 类型系统：值类型、引用类型、可空、枚举、结构体。
3. 面向对象：类、对象、属性、构造函数、继承、接口。
4. 集合与泛型：用泛型表达“类型安全的复用”。
5. 委托与事件：把“发生了什么”和“谁来响应”解耦。
6. LINQ：把集合操作写得更像数据变换。
7. 异常与资源释放：失败路径也要设计。
8. 异步编程：理解 `Task`、`await`、取消和异常传播。
9. Unity C#：把语言特性放进游戏引擎语境里使用。

## 怎么练

- 每学一个语法点，都写一个 20 行以内的小例子。
- 每学一个概念，都问自己：它解决了什么重复、耦合或错误风险？
- 用 Unity 项目反推学习：看到 `MonoBehaviour`、`SerializeField`、`Coroutine`、事件系统时，再回来查对应笔记。
- 不要一开始追求“高级写法”。C# 很多高级特性是为了让大型代码更可维护，不是为了炫技。

## 一个最小练习项目

可以做一个“背包系统”的控制台版本：

- `Item` 表示物品。
- `Inventory` 管理物品列表。
- `IUsable` 表示可使用物品。
- `OnItemAdded` 用事件通知 UI。
- 用 LINQ 查询稀有物品。
- 用 JSON 保存和读取背包。

这个小项目能覆盖 C# 的大部分核心概念，而且很贴近 Unity 游戏开发。

## 常见误区

- 把 `class` 当成“数据袋”。类不只是字段集合，也应该封装行为和不变量。
- 到处使用 `public` 字段。普通 C# 项目里更推荐属性；Unity 里可以用 `[SerializeField] private` 暴露 Inspector。
- 看到 LINQ 就全都用 LINQ。热路径、每帧 Update、GC 敏感场景要谨慎。
- 把协程当多线程。Unity 协程是分帧执行，不等于后台线程。
- `async void` 到处飞。除了事件处理器，通常应返回 `Task` 或 `Task<T>`。
