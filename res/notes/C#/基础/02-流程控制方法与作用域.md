# 流程控制、方法与作用域
> 2026-06 | C# · 基础 · 方法

流程控制决定代码“走哪条路”，方法决定代码“如何分块”。写 C# 时，应该尽早养成把复杂逻辑拆成小方法的习惯。

## if / else

```csharp
if (hp <= 0)
{
    Die();
}
else if (hp < 30)
{
    PlayLowHpWarning();
}
else
{
    ContinueBattle();
}
```

判断条件应该尽量表达业务含义。如果条件越来越长，考虑提取方法：

```csharp
if (IsInDanger(hp, enemyDistance))
{
    Escape();
}
```

## switch

适合根据离散状态分支：

```csharp
switch (state)
{
    case PlayerState.Idle:
        PlayIdle();
        break;
    case PlayerState.Attack:
        PlayAttack();
        break;
    default:
        PlayFallback();
        break;
}
```

如果分支越来越多，而且每个分支都很复杂，说明可能需要状态模式或策略类。

## for / foreach / while

`for` 适合需要索引：

```csharp
for (int i = 0; i < enemies.Count; i++)
{
    enemies[i].Tick();
}
```

`foreach` 适合只关心元素：

```csharp
foreach (var enemy in enemies)
{
    enemy.Tick();
}
```

`while` 适合循环次数不确定：

```csharp
while (queue.Count > 0)
{
    Process(queue.Dequeue());
}
```

Unity 热路径中要注意枚举器和分配问题，但现代 C# 与常见集合已经优化了很多。真正遇到性能问题时再用 Profiler 判断。

## 方法

方法应该做一件清楚的事：

```csharp
int CalculateDamage(int attack, int defense)
{
    int raw = attack - defense;
    return Math.Max(raw, 1);
}
```

命名建议：

- 返回 bool 的方法常用 `Is`、`Has`、`Can` 开头。
- 执行动作的方法使用动词。
- 计算结果的方法可用 `Calculate`、`Get`、`Find`。

## 参数与返回值

```csharp
bool TrySpendGold(int cost)
{
    if (gold < cost)
    {
        return false;
    }

    gold -= cost;
    return true;
}
```

`TryXxx` 是 C# 常见命名习惯：尝试做某事，成功返回 `true`，失败返回 `false`。

## 作用域

变量只在声明它的代码块内有效：

```csharp
if (isCritical)
{
    int bonus = 20;
}

// bonus 在这里不可用
```

作用域越小，代码越安全。不要把临时变量提升成字段，除非它确实代表对象状态。

## early return

比起多层嵌套，早返回常常更清楚：

```csharp
void Attack(Target target)
{
    if (target == null) return;
    if (!CanAttack(target)) return;

    DealDamage(target);
    PlayEffect();
}
```

这种写法在 Unity 的输入、碰撞、UI 回调里特别常用。

## 小练习

写一个购买物品方法：

- 金币不足返回失败。
- 背包满返回失败。
- 成功时扣金币并添加物品。
- 用 `TryBuyItem` 命名。
