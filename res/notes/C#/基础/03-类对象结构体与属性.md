# 类、对象、结构体与属性
> 2026-06 | C# · OOP · 类型设计

类和对象是 C# 组织复杂代码的核心。类不只是“字段的集合”，更重要的是把数据和行为放在一起，并保护对象始终处于合法状态。

## 类与对象

类是蓝图，对象是实例：

```csharp
public class Player
{
    public string Name { get; private set; }
    public int Level { get; private set; }

    public Player(string name)
    {
        Name = name;
        Level = 1;
    }

    public void LevelUp()
    {
        Level++;
    }
}
```

创建对象：

```csharp
var player = new Player("Luna");
player.LevelUp();
```

## 字段与属性

字段是存储数据的位置：

```csharp
private int hp;
```

属性是对外暴露数据的接口：

```csharp
public int Hp
{
    get => hp;
    private set => hp = Math.Clamp(value, 0, MaxHp);
}
```

属性可以控制读写权限，也可以在赋值时保护规则。

## 自动属性

简单场景用自动属性：

```csharp
public int Gold { get; private set; }
```

外部能读，只有类内部能改。比公开字段更容易维护。

## 构造函数

构造函数用于建立对象初始状态：

```csharp
public Item(string id, string name, int price)
{
    Id = id;
    Name = name;
    Price = price;
}
```

构造函数不要做太重的事，比如网络请求、复杂 IO。它应该尽量只初始化对象。

## struct 结构体

结构体是值类型，适合小而简单的数据：

```csharp
public struct Damage
{
    public int Value { get; }
    public bool IsCritical { get; }

    public Damage(int value, bool isCritical)
    {
        Value = value;
        IsCritical = isCritical;
    }
}
```

Unity 里的 `Vector3`、`Quaternion`、`Color` 都是结构体。

## class 与 struct 的选择

优先用 `class`，除非满足这些条件：

- 数据很小。
- 表示一个值，而不是有身份的对象。
- 通常不可变。
- 频繁创建时希望减少堆分配。

如果一个类型需要继承、多态、生命周期、共享身份，通常用类。

## 封装

坏味道：

```csharp
player.hp -= 999;
```

更好的方式：

```csharp
player.TakeDamage(999);
```

`TakeDamage` 可以统一处理护盾、死亡、事件、音效，而不是让外部随便改字段。

## Unity 注意点

Unity 中常见写法：

```csharp
[SerializeField] private int maxHp = 100;

public int Hp { get; private set; }
```

这样 Inspector 可配置，但外部代码不能随意改字段。比 `public int maxHp;` 更稳。
