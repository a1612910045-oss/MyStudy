# 类型、变量与字符串
> 2026-06 | C# · 基础 · 类型系统

C# 是静态类型语言。变量在编译时就有明确类型，编译器会帮你检查很多错误。学习 C# 的第一步，不是记住所有关键字，而是理解“类型决定了这个值能做什么”。

## 变量声明

最常见写法：

```csharp
int level = 10;
float speed = 3.5f;
string playerName = "Luna";
bool isAlive = true;
```

也可以用 `var` 让编译器推断类型：

```csharp
var score = 100;       // int
var title = "Knight";  // string
```

`var` 不是动态类型。变量类型依旧在编译期确定，只是不用手写。

## 常见基础类型

- `int`：整数，常用于数量、等级、索引。
- `float`：单精度浮点数，Unity 中位置、速度、时间经常用它。
- `double`：双精度浮点数，普通 .NET 数学计算常见。
- `decimal`：高精度十进制，适合金额，不适合游戏坐标。
- `bool`：布尔值，只有 `true` 和 `false`。
- `char`：单个字符。
- `string`：字符串，本质上是字符序列。

## 值类型与引用类型

C# 类型大体分为值类型和引用类型。

值类型变量保存值本身：

```csharp
int a = 10;
int b = a;
b = 20;

Console.WriteLine(a); // 10
```

引用类型变量保存对象引用：

```csharp
Player p1 = new Player();
Player p2 = p1;
p2.Name = "New Name";

Console.WriteLine(p1.Name); // New Name
```

这件事非常重要。它决定了参数传递、集合存储、对象共享时会不会互相影响。

## 字符串插值

推荐用字符串插值来拼接可读文本：

```csharp
string name = "Luna";
int level = 12;
string text = $"{name} reached level {level}.";
```

比 `name + " reached level " + level` 更清晰，也更容易维护。

## 字符串是不可变的

每次拼接字符串，通常都会产生新字符串：

```csharp
string text = "";
text += "A";
text += "B";
text += "C";
```

少量拼接没问题；大量循环拼接时，使用 `StringBuilder` 更合适：

```csharp
var builder = new StringBuilder();
builder.Append("HP: ");
builder.Append(100);
string result = builder.ToString();
```

Unity 中如果每帧大量拼接 UI 字符串，可能带来额外 GC 压力。

## 可空引用类型

较新的 C# 支持可空引用类型标注：

```csharp
string name = "Luna";
string? optionalName = null;
```

`?` 表示这个引用可能为 `null`。它不是运行时魔法，而是让编译器帮助你发现潜在空引用。

## 小练习

写一个角色状态输出：

```csharp
string name = "Luna";
int hp = 80;
int maxHp = 100;
float percent = hp / (float)maxHp;

Console.WriteLine($"{name}: {hp}/{maxHp} ({percent:P0})");
```
