# 继承、接口与多态
> 2026-06 | C# · OOP · 接口

继承和接口都能表达“某些类型有共同能力”，但它们的使用场景不同。写游戏代码时，组合和接口通常比深继承更灵活。

## 继承

继承表示“是一个”关系：

```csharp
public class Enemy
{
    public virtual void Attack()
    {
        Console.WriteLine("Enemy attack");
    }
}

public class Slime : Enemy
{
    public override void Attack()
    {
        Console.WriteLine("Slime attack");
    }
}
```

`virtual` 表示允许子类重写，`override` 表示正在重写父类方法。

## 多态

父类引用可以指向子类对象：

```csharp
List<Enemy> enemies = new()
{
    new Slime(),
    new Goblin(),
};

foreach (var enemy in enemies)
{
    enemy.Attack();
}
```

调用哪个 `Attack`，取决于运行时对象实际类型。

## 抽象类

抽象类不能直接实例化，适合提供共同基础：

```csharp
public abstract class Skill
{
    public string Name { get; }

    protected Skill(string name)
    {
        Name = name;
    }

    public abstract void Cast();
}
```

抽象类可以有字段、构造函数、已实现方法。

## 接口

接口表达“能做什么”：

```csharp
public interface IDamageable
{
    void TakeDamage(int amount);
}

public interface IInteractable
{
    void Interact(Player player);
}
```

一个类可以实现多个接口：

```csharp
public class Chest : IInteractable, IDamageable
{
    public void Interact(Player player) { }
    public void TakeDamage(int amount) { }
}
```

## 什么时候用接口

- 你关心能力，而不是继承层级。
- 多种类型都能响应同一操作。
- 需要解耦调用方和具体实现。
- 想让测试更容易替换依赖。

例如攻击系统只需要知道目标能受伤：

```csharp
void Hit(IDamageable target)
{
    target.TakeDamage(10);
}
```

它不需要知道目标是怪物、木箱还是玩家。

## 组合优先

深继承容易变脆：

```text
Enemy
  FlyingEnemy
    FireFlyingEnemy
      BossFireFlyingEnemy
```

组合更灵活：

```text
Enemy + FlyMovement + FireAttack + BossStats
```

Unity 的组件模型本身就是组合思维：一个 GameObject 通过多个 Component 获得能力。

## 常见建议

- 不要为了复用两行代码就继承。
- 继承层级超过 2-3 层时要警惕。
- 接口命名通常以 `I` 开头，如 `IUsable`、`ISaveable`。
- 接口应该小而清楚，不要做成“万能接口”。
